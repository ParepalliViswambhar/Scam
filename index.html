Exp-1 — EC2 Access

a) Connect using PuTTY (.ppk)

  1. Create an EC2 with a key pair of .ppk, copy the IP of it.
  2. Open PuTTY, paste the IP.
  3. Left side: Connection → SSH → Auth → Credentials, browse the .ppk and click Open.
  4. If Linux login as: ec2-user, if Ubuntu login as: ubuntu.   Test: whoami (console)

b) Convert .pem to .ppk

Convert .pem file to .ppk using PuTTYGen, save as private key file, then follow the same process as above.

c) RDP (Windows EC2)

  1. Create an EC2 — in security groups add RDP in the custom TCP, select OS: Windows.
  2. Click on Connect → using RDP client → click Get Password → upload the .pem file to decrypt.
  3. Click the Windows symbol and search RDP, paste the IP of the EC2.
  4. Login as Administrator, password: decrypted password.
  5. You can see the new system.

Exp-2 — Volumes & Snapshots

Volumes

a) Attach a Volume

  1. Create an EC2, select an AZ.
  2. Create a volume with the same AZ, size anything.
  3. Attach volume to the running EC2, device name: /dev/sdf.
  4. Open EC2 console and run lsblk.

b) Format and Mount the Volume

After attaching and connecting to EC2:

sudo su
lsblk -fs
fdisk /dev/nvme1n1    (m, n, p, , , , w)
lsblk
mkfs.xfs /dev/nvme1n1p1
mkdir /mnt/vish
mount /dev/nvme1n1p1 /mnt/vish
df -h
blkid /dev/nvme1n1p1
(COPY UUID)
nano /etc/fstab
(inside that, last line add) =UUID=<UUID-value>   /mnt/vish   xfs   defaults,nofail   0   0
mount -a
reboot and check for files

c) Move Volume to Another EC2

Create another EC2 with the same AZ, detach the volume from the old EC2 and attach to the new one.

mkdir /mnt/vish
mount /dev/nvme1n1p1 /mnt/vish
lsblk -fs

Snapshots

  1. For a given volume you created, click on Actions → Create Snapshot.
  2. From the Snapshots section, click on Copy Snapshot and select some region.
  3. Switch to that region and create a volume for it from Actions.
  4. Attach an EC2 in that region to the created volume with the same AZ. (run lsblk if needed)

Exp-3 — EFS (Elastic File System)

  1. Create 2 EC2s with 2 different availability zones and include NFS in the custom TCP.
  2. Create an EFS with only 2 availability zones.
  3. Open consoles of both EC2s.
  4. Run the following commands on both:
sudo su
mkdir efs
yum install -y amazon-efs-utils
(connect the efs with dns) efs → connect → dns → paste in console
cd efs
in any 1 do: nano h.txt and put text
(it shows in the other EC2)

EXP-4 — S3

a) Public Access to S3 Object

  1. Create an S3 bucket, allow ACLs under ownership.
  2. Block all public access — uncheck that (very important).
  3. Enable versioning (better, because others need it).
  4. After creation: Bucket → Permissions → ACL → Public Access → right side tick.
  5. Upload any image, click on it → Permissions → ACL → Public Access → left and right tick.
  6. Properties → Copy link → open in incognito.

b) Versioning

  1. Enable versioning (compulsory).
  2. Upload a text file, change it locally, and upload again.
  3. Buckets → Objects → click Show Versions → you can see both versions.
  4. Delete the latest version and the old version still shows up.

c) Cross Region Replication

  1. Create 1 more bucket in a different region with the same process including versioning enabled.
  2. Go to first bucket → Management → Replication Rules.
  3. Status = Enable, Source bucket = Apply to all objects, Destination bucket = select the created one, IAM Role = Lab Role. Replicate existing objects → No (this option comes after creating).
  4. Upload any file in bucket 1 — it will automatically show in the 2nd bucket.

d) Static Website Hosting

  1. Upload index.html and error.html into S3 following the normal process including ACL settings and allowing static hosting. Put index and error filenames in the fields.
  2. Click on index.html → Permissions → Everyone → both tick. Same with error.html.
  3. Bucket → Properties → scroll to Static Website Hosting → copy URL → check.

EXP-5 — VPC

  1. Create a VPC, CIDR: 10.0.0.0/16.
  2. Create subnets:
    • Public: AZ-east-1a, subnet CIDR: 10.0.1.0/24
    • Private: AZ-east-1a, subnet CIDR: 10.0.2.0/24
  3. Create an Internet Gateway, attach the VPC (shown after creation).
  4. Create Route Tables:
    • Public: Create → Actions → Edit Routes → Add Route → Destination: 0.0.0.0/0, Internet Gateway (custom made) → Actions → Edit Subnet Associations → add the public subnet.
    • Private: Create → Actions → Edit Subnet Associations → add the private subnet.
  5. Public subnet → Actions → Edit Subnet Settings → Auto-assign IP (tick).
  6. Create a security group, add the custom VPC under creation, inbound rules: SSH, HTTP, HTTPS.
  7. Create an EC2 for the public subnet: Edit Network → VPC (custom), Subnet: public, select existing security group: custom made.
  8. Do the same for the EC2 of the private subnet.
  9. Connecting to both: public subnet EC2 will connect, private will not.
